# Concise Planning

## 目的
ユーザーのリクエストを、**単一の実行可能な計画（Atomic Checklist）**に変換します。

## ワークフロー

1.  **コンテキスト把握**: README、ドキュメント、関連コードを読み、制約を確認。
2.  **最小限の対話**: 質問はブロッキングな場合のみ1〜2個に留める。仮定を置いて進める。
3.  **プラン作成**: 以下のテンプレートを使用。

## プランテンプレート

```markdown
# Plan

<アプローチの概要（1-3行）>

## Scope
- In: 実装範囲
- Out: 対象外

## Action Items
[ ] <Step 1: 調査・設計>
[ ] <Step 2: 実装（コンポーネントA）>
[ ] <Step 3: 実装（ロジックB）>
[ ] <Step 4: テスト・検証>
[ ] <Step 5: コミット・クリーンアップ>

## Open Questions
- <未解決事項（あれば）>
```

## ガイドライン
- **Atomic**: 各ステップは単一の論理的作業単位にする。
- **Verb-first**: 「〜を追加する」「〜をリファクタリングする」のように動詞から始める。
- **Concrete**: 可能な限り具体的なファイル名やモジュール名を記載する。
