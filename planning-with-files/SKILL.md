# Planning with Files

複雑なタスクにおいて、メモリ（コンテキスト）だけでなくディスク（Markdownファイル）を「外部記憶」として活用する手法です。

## 3つの重要なファイル
これらは**プロジェクトルート**に作成します（スキルフォルダではありません）。

| ファイル | 目的 | 更新タイミング |
|---|---|---|
| `task_plan.md` | フェーズ、進捗、決定事項の管理 | 各フェーズ終了時 |
| `findings.md` | 調査結果、発見、学び | 新しい発見があった時 |
| `progress.md` | 作業ログ、エラー、試行錯誤 | セッション中随時 |

## ルール

1.  **まずプラン作成**: 複雑な作業は `task_plan.md` の作成から始める。
2.  **2アクション・ルール**: 2回以上の調査・表示を行ったら、重要な発見を `findings.md` に書き出す。
3.  **決定前に読む**: 大きな決定をする前にプランを読み返す。
4.  **失敗を記録**: エラーや失敗したアプローチはログに残し、繰り返さない。

## テンプレート

`skills/planning-with-files/templates/` 配下のテンプレートを参考に、プロジェクトルートに作成してください。

## いつ使うか？
- **使う**: 複数ステップのタスク、調査タスク、新規プロジェクト構築。
- **使わない**: 単純な質問、単一ファイルの修正、簡単な検索。
